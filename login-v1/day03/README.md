## 今天的屎味

### 🔴 `Math.random()` 生成重置令牌

```javascript
const token = Math.random().toString(36).substring(2);
```

`Math.random()` **不是密码学安全的随机数生成器**。攻击者如果可以预测随机数种子，就能伪造重置链接。

应该用 `crypto.randomBytes()` 或专门的 token 库——但「先跑起来再说」的时候，谁会去想这个？

### 🔴 重置令牌没有过期时间

生成的 token **永久有效**。用户今天申请了重置，三个月后这个链接还能用。

攻击者只要能拿到一个历史重置链接，就能修改对应用户的密码。

### 🔴 泄漏「邮箱是否已注册」

```javascript
if (!user) {
  res.json({ success: false, message: '该邮箱未注册' });
}
```

攻击者可以枚举邮箱列表——拿一批邮箱去调 `forgot-password`，看哪个返回「未注册」就是真实用户。

安全做法是统一返回「如果邮箱存在，重置链接已发送」，不告诉对方邮箱是否存在。但这么改了，调试不方便啊，所以「先这样吧」。

### 🔴 「已发送邮件」是假的

```javascript
console.log('重置链接: http://localhost:3000/reset-password?token=...');
res.json({ success: true, message: '重置链接已发送到您的邮箱' });
```

实际上只是打了个 log。要是以后真接邮件服务，这里还得改——**但到时候谁会记得这个 TODO？**

### 🔴 SQL 注入点 +2

`/forgot-password` 和 `/reset-password` 继续用字符串拼接，攻击面又扩大了。
